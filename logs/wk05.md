## What I learned

### Spotting the generic AI look

In the MP1 showcase, Dr. Li walked through the checklist of things AI defaults to: rectangular cards everywhere, large rounded corners, generic fonts, warm paper backgrounds, and inconsistent pages across a multi-page site. The fix is to be specific in prompts by naming the font, the colors, and the corner style rather than saying "make it look nice."

### JavaScript is about making pages react

The dark-mode toggle shown in class was a useful first preview: clicking a button adds or removes a CSS class, and that class carries all the dark colors. That is the core idea. JavaScript listens for something to happen and then changes the page in response. Everything in `js-demo.html` follows that same pattern.

### Reading code before changing it

Before changing anything AI gives you, find the `<script>` tag, every `querySelector`, every `addEventListener`, and every place the code changes the page. Read it first, test it, then change it.

### JS Demo don't-know list (from reading `js-demo.html`)

**1. How `Math.random()` generates a random color**

The demo uses `Math.random()` to pick a background color, but I did not understand how a random decimal between 0 and 1 becomes a color. It multiplies the random number by 256 three times to get random red, green, and blue values, then combines them into an `rgb(r, g, b)` string. I would not have known to write this pattern myself.

**2. How `classList.toggle()` works to show and hide elements**

The hide-and-seek demo uses `classList.toggle('hidden')` to make text disappear and reappear. I knew `classList.add` and `classList.remove` existed, but I did not know `toggle` existed. It checks whether the class is already present, removes it if it is, adds it if it is not. One method replacing two separate if/else branches.

**3. What `style.transform` does to make the mailbox grow on hover**

The mailbox uses `style.transform = 'scale(1.5)'` on mouseenter and resets on mouseleave. I did not know that `transform: scale()` was accessible through JavaScript as `style.transform`, or that it could be set directly on an element without touching the CSS file.
