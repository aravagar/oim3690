## What I learned

- **Arrays + loops**: the render-from-data pattern (an array, a loop, one element per entry) is the same shape underneath a to-do list, a feed, a search result list, or a playlist. Once you see the loop and the `if` underneath, it stops feeling like magic.
- **`for...of` vs classic `for`**: `for...of` is cleanest when I just need each value; the classic `for (let i = 0; ...)` is worth reaching for specifically when I need the index.
- **`push` vs `unshift`**: `push` adds to the end of an array, `unshift` adds to the start. Small distinction, but it matters for anything ordered (like a playlist or a message feed).
- **POST requests**: sending data _to_ an API uses the same `fetch`/`await` pattern as reading data, just with a `method: "POST"`, a `body`, and headers like `X-Token` for authentication.
