## What I learned

### Responsive design and forms

- The viewport meta tag is what makes a site actually respond on a phone instead of rendering a shrunk-down desktop page.
- A fixed pixel width on a container will overflow a narrow screen and force a horizontal scrollbar. Removing the fixed width and letting Flexbox or Grid handle it fixes that.
- Media queries like `@media (max-width: 768px)` let me change the layout at specific breakpoints, for example collapsing a horizontal nav into a stacked one.
- Forms need a `<label>` tied to each input with `for` and `id` for accessibility, and validation attributes like `required` and input types like `email` and `tel` do real work before anything reaches a server.
- Image background matters. A logo saved with a black background shows up as a black box on a white header. The quick fix I used was taking a screenshot of the logo on a white background and using that image instead.
