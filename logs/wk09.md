## What I learned

### An API you depend on can disappear while you are building on it

REST Countries v3.1 went fully dead partway through the build. The replacement, v5, requires an API key, and a key is useless to me here: GitHub Pages serves static files with no server, so there is nowhere to hide it. That ruled the whole provider out and I moved to `countries.dev`.

The takeaway is a checklist item now: before I commit to an API for a static site, confirm it is current, confirm it needs no key, and confirm it sends CORS headers.

### Bulk fetch beats a call per item

Both World Bank GDP data and Frankfurter rates are pulled once for every country in a single request, then read from memory. My first instinct was a fetch per country, which would have meant ~200 requests and a page that crawls.

### Leaflet has two defaults that will bite you

```js
L.tileLayer(url, { noWrap: true }).addTo(map);
map.setMaxBounds([[-90, -180], [90, 180]]);
```

Without `noWrap`, tiles repeat and you get an infinite row of Earths. Separately, Leaflet's own controls sit at a z-index around 1000, so my detail panel at z-index 50 had the map bleeding through it. I had to raise the panel above that.

### The browser and GitHub Pages will both serve you stale JavaScript

I spent a while debugging code that was already fixed. The fix is a cache-busting query string on the script tag:

```html
<script src="js/app.js?v=7"></script>
```

Bump the number, browser treats it as a new file.

### What Lighthouse actually rewards

- `loading="lazy"` on an above-the-fold image **hurts** you. It delays the largest paint instead of helping it.
- `role="button"` on an `<article>` is an ARIA mismatch and gets flagged.
- Skipping from `h1` to `h3` breaks heading order.
- CSP, HSTS and COOP warnings are **unfixable on GitHub Pages**, since setting response headers needs a server. Worth knowing so I stop trying to fix them.

### localStorage, from the 7/13 class

It "comes from nowhere," meaning I never declared it, which is the tell that the browser hands it to me for free. It stores strings only, so objects go through `JSON.stringify` on the way in and `JSON.parse` on the way out. It lives in one browser on one machine, so it is never the source of truth.

### Scope is what you leave out on purpose

The line from the FP kickoff that changed how I wrote my proposal. I added a "what this does not do (for now)" section, which is not required, specifically so the project could not quietly grow past what I can finish.

## AI interactions

### Interaction 1

- **What I asked**: My country grid rendered blank with no obvious console error. I asked AI to work out why the fetch was returning nothing.
- **What AI gave me**: It identified that REST Countries v3.1 had been retired, explained that v5 now requires an API key, and recommended `countries.dev` as a keyless, CORS-enabled replacement.
- **What I kept / changed / rejected**: Kept the diagnosis and the replacement. I did **not** let it rewrite the fetch layer straight away. I opened the `countries.dev` endpoint in the browser first to see the actual JSON shape, because the field names differ from REST Countries and I wanted to map them myself rather than trust that it had guessed right.

### Interaction 2

- **What I asked**: Why my detail panel opened but the map showed through it, on top of the panel content.
- **What AI gave me**: It explained Leaflet assigns its own internal controls z-index values up to about 1000, so my panel at 50 was underneath them, and told me to raise the panel and its overlay above that.
- **What I kept / changed / rejected**: Kept it exactly. This is the kind of thing I would never have found by reading my own CSS, because nothing in my code sets 1000. The number comes from inside the library.

### Interaction 3

- **What I asked**: To draft `PROPOSAL.md` for the final project.
- **What AI gave me**: A first draft built around a six-stage sales-only pipeline (New, Meeting, Quoting, Quoted, Won, Lost).
- **What I kept / changed / rejected**: **Rejected the structure.** A sales-only pipeline does not describe the problem Studioforma actually has, which is that a project falls off a cliff between the sales sheet and the production floor. I had it rework the whole thing into one continuous lead-to-production pipeline, and add stage management so the stage names can be edited later instead of being frozen by a guess I made in July. I also cut every reference to Google Sheets, since the tool is standalone and mentioning the sheet made the scope read as an integration project.
